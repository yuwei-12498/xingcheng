# 结果页、地图驾驶舱与模型信息上屏设计

**目标**

在不回退现有功能的前提下，恢复结果页的高信息密度视觉方案，并把模型与路线计算产出的信息更完整地展示出来：包括方案推荐理由、首段出发信息、按天查看、多源 POI 状态提示、地图空间解读与社区联动状态。

**范围**

- 前端结果页 `F:\dachuang\frontend\src\views\Result.vue`
- 地图卡 `F:\dachuang\frontend\src\components\itinerary\ItineraryMapCard.vue`
- 结果页展示辅助工具 `F:\dachuang\frontend\src\utils\resultUi.js`
- 对应前端单测

**视觉方向**

- 延续当前社区页的蓝白玻璃化视觉体系，不回退到旧的普通卡片堆叠界面。
- 结果页升级为“路线驾驶舱”结构：左侧地图/右侧时间线，下方海报导出，顶部双栏 Hero。
- 地图卡升级为“可读地图面板”，而不是只有一张 Leaflet 底图。

**交互结构**

1. **Hero 双栏**
   - 主区域：路线标题、摘要、首段出发说明、关键信息胶囊、执行脚注。
   - 侧区域：推荐理由、主操作（换一版路线）、常用操作（收藏/发布/海报/社区帖子）、页面跳转。

2. **方案对比**
   - 保留现有候选方案切换能力。
   - 仅当存在 2 个及以上方案时显示。

3. **结果驾驶舱**
   - 左侧：地图驾驶舱卡片。
   - 右侧：时间线面板，支持按天切换。
   - 时间线卡片内展示来源类型、AI 温馨提示、周边服务、路上耗时、交通方式、距离与花费。

4. **海报**
   - 海报沿用当前生成链路，但改为基于当前显示的日程片段 `displayNodes`，和按天查看保持一致。

**模型信息上屏策略**

- 用 `departureTravelTime / departureTransportMode / departureDistanceKm` 生成“首段出发说明”。
- 用 `recommendReason / tips / highlights / alerts / statusNote / nearby*` 生成更完整的结果页解释层。
- 用路线点位的城区、类别、时段信息在地图卡内生成空间解读文案与指标卡。

**技术方案**

- 把“结果页展示逻辑”集中在 `resultUi.js`：
  - 统计卡构造
  - 操作分组构造
  - Hero 文案与胶囊构造
  - 按天切分逻辑
  - 路上信息格式化
- `Result.vue` 负责组合与渲染。
- `ItineraryMapCard.vue` 负责地图渲染与地图驾驶舱文案。

**验证策略**

- 使用 Vitest 锁定结果页高级结构、按天切换、地图驾驶舱结构、结果页辅助工具行为。
- 最终跑前端单测与 `vite build`。
